DataMateApps Permission Scopes

DataMateApps uses the following permission scopes:

https://www.googleapis.com/auth/gmail.modify: This scope allows DataMateApps to read, compose, send, and permanently delete emails in your Gmail account. The app uses this to manage email drafts, send emails, and delete emails as part of its email automation features, such as sending data reports or clearing outdated drafts. A narrower scope would not suffice because the app requires full email management to function as intended.

https://www.googleapis.com/auth/gmail.compose: This scope allows DataMateApps to create and modify email drafts in Gmail. The app uses this to prepare emails with pre-filled data from your Google Sheets or Drive files for user review before sending. A narrower scope would not suffice because draft creation and modification are essential for the app’s email composition functionality.

https://www.googleapis.com/auth/gmail.send: This scope allows DataMateApps to send emails on your behalf. The app uses this to send emails directly from your Gmail account, such as automated reports or notifications generated from your data. A narrower scope would not suffice because sending emails is a core feature of the app.

https://www.googleapis.com/auth/gmail.readonly: This scope allows DataMateApps to read email content and metadata in your Gmail account. The app uses this to retrieve email data for processing, such as extracting information from incoming emails to populate Google Sheets. A narrower scope would not suffice because read access is necessary for email-based data integration.

https://www.googleapis.com/auth/drive: This scope allows DataMateApps to see, edit, create, and delete all your Google Drive files. The app uses this to store and manage files, such as uploaded documents or generated reports, and to collect signatures stored in Drive. A narrower scope would not suffice because the app requires full Drive access to handle file operations and ensure seamless integration.

https://www.googleapis.com/auth/spreadsheets: This scope allows DataMateApps to see, edit, create, and delete all your Google Sheets spreadsheets. The app uses this to create sheets, populate them with data, and apply formulas to save and organize records. A narrower scope would not suffice because the app’s core functionality depends on dynamic spreadsheet creation and editing.

https://www.googleapis.com/auth/script.send_mail: This scope allows DataMateApps to send emails as you via Google Apps Script. The app uses this to send automated emails, such as confirmations or alerts, directly from the script environment. A narrower scope would not suffice because this scope is required for script-based email sending independent of Gmail API restrictions.

https://www.googleapis.com/auth/script.external_request: This scope allows DataMateApps to connect to external services. The app uses this to fetch data from third-party APIs or services to enhance its functionality, such as integrating external data into your sheets or files. A narrower scope would not suffice because external connectivity is essential for the app’s data processing capabilities.

https://www.googleapis.com/auth/userinfo.email openid: This scope allows DataMateApps to access your email address and basic profile information. The app uses this to authenticate users and personalize email communications, ensuring emails are sent from and associated with the correct account. A narrower scope would not suffice because email identification is required for user authentication and functionality.

AddressBlock Permission Scopes

AddressBlock uses the following permission scopes:

https://www.googleapis.com/auth/spreadsheets: This scope allows AddressBlock to see, edit, create, and delete all your Google Sheets spreadsheets. The app uses this to generate spreadsheets containing address block data, apply formulas, and organize address information. A narrower scope would not suffice because spreadsheet creation and editing are core to the app’s address block generation.

https://www.googleapis.com/auth/script.external_request: This scope allows AddressBlock to connect to external services. The app uses this to retrieve address-related data from third-party APIs, such as geocoding services, to enhance address block accuracy. A narrower scope would not suffice because external data integration is essential for the app’s functionality.

https://www.googleapis.com/auth/script.container.ui: This scope allows AddressBlock to display and run third-party web content in prompts and sidebars within Google applications. The app uses this to show interactive UI elements, such as address input forms or previews, directly in Google Sheets or Docs. A narrower scope would not suffice because the app relies on custom UI for user interaction.
